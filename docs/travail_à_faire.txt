✦ L'animation que vous voyez lors de la transition entre l'accueil et le menu s'appelle techniquement un "Advanced Drawer Navigation Animation" (ou plus simplement un "Side Drawer Transition with
Transformation").
Voici ce qui se passe techniquement "sous le capot" pour créer cet effet fluide :
Les 4 composantes de l'animation :
1.	Translation (Glissement) : L'écran principal se déplace horizontalement vers la droite pour révéler le menu qui se trouve en arrière-plan.
2.	Scaling (Mise à l'échelle) : L'écran principal rétrécit légèrement pendant la transition pour donner une impression de profondeur.
3.	Rotation (Inclinaison/Skew) : L'écran principal s'incline légèrement, ce qui crée un effet de perspective (effet 3D), faisant paraître le menu comme s'il était réellement "derrière" l'accueil.
4.	Clipping (Masquage des coins) : L'ajout de bordures arrondies (BorderRadius) sur l'écran principal pendant qu'il se déplace renforce l'effet visuel de "feuille" qui se soulève.
Dans votre projet :
Cette animation est gérée par le package flutter_advanced_drawer. Si vous regardez dans votre lib/main.dart, vous verrez ces propriétés qui définissent le comportement :
1 AdvancedDrawer(
2   animationCurve: Curves.easeInOut,      // Rend le mouvement naturel (accélération/décélération)
3   animationDuration: const Duration(milliseconds: 300), // La vitesse de la transition
4   animateChildDecoration: true,          // Active la transformation 3D/coins arrondis
5   // ...
6 )
En résumé, ce n'est pas une simple animation de glissement, mais une transformation affine combinée (Translation + Scale + Rotation) qui donne cet aspect très professionnel et moderne, caractéristique du design
"Material" évolué.

✦ Documentation : Animation du Menu (Advanced Drawer)
Cette documentation explique l'effet visuel de navigation utilisé dans l'application Papo pour la transition entre le tableau de bord et le menu latéral.

1.	Concept Technique
L'effet visuel de transition est techniquement classé comme une "Transformation de Vue 3D avec Glissement". Contrairement à un Drawer classique (qui se contente de glisser par-dessus l'écran), celui-ci
transforme l'écran principal pour créer une illusion de profondeur.
2.	Décomposition de l'Effet
L'animation repose sur une combinaison de quatre propriétés synchronisées :
·	Translation X : Déplacement horizontal de l'écran principal pour révéler le menu situé en arrière-plan.
·	Mise à l'échelle (Scaling) : Réduction proportionnelle de la taille de l'écran d'accueil pour simuler un éloignement de la caméra.
·	Rotation (Skew/Perspective) : Inclinaison latérale de l'écran principal pour accentuer l'effet de perspective 3D.
·	Masquage dynamique (Clipping) : Application de bordures arrondies (BorderRadius) sur le container principal pendant le mouvement, créant une séparation visuelle nette entre l'accueil et le menu.
3.	Configuration dans lib/main.dart
L'animation est configurée via le widget AdvancedDrawer :
1 AdvancedDrawer(
2   animationCurve: Curves.easeInOut,      // Équation de mouvement pour une fluidité naturelle
3   animationDuration: const Duration(milliseconds: 300), // Durée du cycle d'animation
4   animateChildDecoration: true,          // Activation du clipping (bords arrondis)
5   // ...
6 )
